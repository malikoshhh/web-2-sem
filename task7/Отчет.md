## 1. XSS (Cross-Site Scripting)

### Описание уязвимости
XSS позволяет злоумышленнику внедрить вредоносный JavaScript код через пользовательский ввод, который затем выполняется в браузере других пользователей.

### Найденные уязвимости

#### До исправления (form.php):
```php
// Уязвимый код - прямой вывод без экранирования
<input name="name" value="<?php echo $name; ?>">
<input name="email" value="<?php echo $email; ?>">
<input name="year" value="<?php echo $year; ?>">
<textarea name="bio"><?php echo $bio; ?></textarea>
```

**Проблема:** Если пользователь введет `<script>alert('XSS')</script>` в поле имени, скрипт выполнится.

#### После исправления (form.php):
```php
// Безопасный код - все выводы экранируются
<input name="name" value="<?php echo htmlspecialchars($name ?? '', ENT_QUOTES, 'UTF-8'); ?>">
<input name="email" value="<?php echo htmlspecialchars($email ?? '', ENT_QUOTES, 'UTF-8'); ?>">
<input name="year" value="<?php echo htmlspecialchars($year ?? '', ENT_QUOTES, 'UTF-8'); ?>">
<textarea name="bio"><?php echo htmlspecialchars($bio ?? '', ENT_QUOTES, 'UTF-8'); ?></textarea>
```

#### До исправления (admin.php):
```php
// Уязвимый код
echo '<td>' . $row['name'] . '</td>';
echo '<td>' . $row['email'] . '</td>';
echo '<td>' . $row['bio'] . '</td>';
```

#### После исправления (admin.php):
```php
// Безопасный код
echo '<td>' . htmlspecialchars($row['name'], ENT_QUOTES, 'UTF-8') . '</td>';
echo '<td>' . htmlspecialchars($row['email'], ENT_QUOTES, 'UTF-8') . '</td>';
echo '<td>' . htmlspecialchars($row['bio'], ENT_QUOTES, 'UTF-8') . '</td>';
```

### Методы защиты
1. **htmlspecialchars()** - экранирует специальные HTML символы
   - `ENT_QUOTES` - экранирует и одинарные, и двойные кавычки
   - `UTF-8` - правильная кодировка для многоязычных данных

2. **filter_var()** - валидация email
```php
// save.php
if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
    $errors[] = 'Введите корректный email.';
}
```

3. **Whitelist валидация** - для ограниченных значений
```php
// save.php - валидация языков программирования
$allowed_languages = ['php', 'javascript', 'python', 'java', 'csharp', 'ruby'];
$languages = isset($_POST['languages']) ? (array)$_POST['languages'] : [];
$languages = array_intersect($languages, $allowed_languages);
```

## 2. SQL Injection

### Описание уязвимости
SQL Injection позволяет злоумышленнику выполнить произвольные SQL запросы через пользовательский ввод, что может привести к утечке, изменению или удалению данных.

### Найденные уязвимости

#### До исправления (save.php):
```php
// КРИТИЧЕСКИ УЯЗВИМЫЙ КОД - прямая подстановка переменных
$stmt = $db->query("INSERT INTO users (name, email, year, gender, limbs, bio, languages) 
                    VALUES ('$name', '$email', $year, '$gender', '$limbs', '$bio', '$languages_str')");
```

**Проблема:** Злоумышленник может ввести в поле email:
```
test@test.com'); DROP TABLE users; --
```

#### После исправления (save.php):
```php
// Безопасный код - prepared statements
$stmt = $db->prepare("INSERT INTO users (name, email, year, gender, limbs, bio, languages) 
                      VALUES (:name, :email, :year, :gender, :limbs, :bio, :languages)");
$stmt->bindParam(':name', $name, PDO::PARAM_STR);
$stmt->bindParam(':email', $email, PDO::PARAM_STR);
$stmt->bindParam(':year', $year, PDO::PARAM_INT);
$stmt->bindParam(':gender', $gender, PDO::PARAM_STR);
$stmt->bindParam(':limbs', $limbs, PDO::PARAM_INT);
$stmt->bindParam(':bio', $bio, PDO::PARAM_STR);
$stmt->bindParam(':languages', $languages_str, PDO::PARAM_STR);
$stmt->execute();
```

#### До исправления (admin.php):
```php
// Уязвимый код
$stmt = $db->query("SELECT * FROM users WHERE id = " . $_GET['delete']);
$stmt = $db->query("DELETE FROM users WHERE id = " . $_GET['delete']);
```

#### После исправления (admin.php):
```php
// Безопасный код
$stmt = $db->prepare("SELECT * FROM users WHERE id = :id");
$stmt->bindParam(':id', $id, PDO::PARAM_INT);
$stmt->execute();

$stmt = $db->prepare("DELETE FROM users WHERE id = :id");
$stmt->bindParam(':id', $id, PDO::PARAM_INT);
$stmt->execute();
```

#### До исправления (edit.php):
```php
// Уязвимый код
$stmt = $db->query("SELECT * FROM users WHERE id = " . $_GET['id']);
```

#### После исправления (edit.php):
```php
// Безопасный код
$stmt = $db->prepare("SELECT * FROM users WHERE id = :id");
$stmt->bindParam(':id', $id, PDO::PARAM_INT);
$stmt->execute();
```

### Методы защиты
1. **Prepared Statements** - параметризованные запросы
   - SQL код отделен от данных
   - Драйвер БД автоматически экранирует параметры

2. **bindParam()** - типизированная привязка параметров
   - `PDO::PARAM_INT` - для целых чисел
   - `PDO::PARAM_STR` - для строк

3. **Валидация типов данных**
```php
// Проверка, что ID - число
$id = filter_input(INPUT_GET, 'id', FILTER_VALIDATE_INT);
if ($id === false || $id === null) {
    die('Некорректный ID');
}
```

## 3. CSRF (Cross-Site Request Forgery)

### Описание уязвимости
CSRF позволяет злоумышленнику выполнить действия от имени авторизованного пользователя без его ведома.

### Найденные уязвимости

#### До исправления:
```php
// admin.php - удаление через GET без защиты
if (isset($_GET['delete'])) {
    $stmt = $db->query("DELETE FROM users WHERE id = " . $_GET['delete']);
}

// edit.php - сохранение без токена
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Прямое сохранение данных
}
```

**Проблема:** Злоумышленник может создать страницу:
```html
<img src="http://yoursite.com/admin.php?delete=1">
```
При открытии этой страницы авторизованным администратором, пользователь с ID=1 будет удален.

#### После исправления:

**Создан security.php:**
```php
<?php
/**
 * Функции безопасности для защиты от CSRF атак
 */

/**
 * Генерирует CSRF токен и сохраняет в сессии
 */
function generateCsrfToken() {
    if (session_status() === PHP_SESSION_NONE) {
        session_start();
    }
    
    if (empty($_SESSION['csrf_token'])) {
        $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
    }
    
    return $_SESSION['csrf_token'];
}

/**
 * Проверяет CSRF токен
 */
function verifyCsrfToken($token) {
    if (session_status() === PHP_SESSION_NONE) {
        session_start();
    }
    
    if (empty($_SESSION['csrf_token'])) {
        return false;
    }
    
    return hash_equals($_SESSION['csrf_token'], $token);
}

/**
 * Получает CSRF токен из POST или GET
 */
function getCsrfTokenFromRequest() {
    return $_POST['csrf_token'] ?? $_GET['csrf_token'] ?? '';
}
```

**Использование в формах (form.php, edit.php):**
```php
<?php
require_once 'security.php';
$csrf_token = generateCsrfToken();
?>

<form action="/web-4-sem/task3/save.php" method="POST">
    <input type="hidden" name="csrf_token" value="<?php echo htmlspecialchars($csrf_token, ENT_QUOTES, 'UTF-8'); ?>">
    <!-- остальные поля -->
</form>
```

**Проверка токена (save.php, edit.php):**
```php
<?php
require_once 'security.php';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $token = getCsrfTokenFromRequest();
    
    if (!verifyCsrfToken($token)) {
        die('CSRF токен недействителен. Обновите страницу и попробуйте снова.');
    }
    
    // Обработка данных
}
```

**Защита операций в admin.php:**
```php
// Удаление только через POST с токеном
if ($_SERVER['REQUEST_METHOD'] === 'POST' && isset($_POST['action'])) {
    $token = getCsrfTokenFromRequest();
    
    if (!verifyCsrfToken($token)) {
        $error = 'CSRF токен недействителен.';
    } else {
        if ($_POST['action'] === 'delete') {
            $id = filter_input(INPUT_POST, 'id', FILTER_VALIDATE_INT);
            // Удаление
        }
    }
}

// В HTML - форма для удаления
<form method="POST" style="display:inline;" onsubmit="return confirm('Удалить пользователя?');">
    <input type="hidden" name="csrf_token" value="<?php echo htmlspecialchars($csrf_token, ENT_QUOTES, 'UTF-8'); ?>">
    <input type="hidden" name="action" value="delete">
    <input type="hidden" name="id" value="<?php echo $row['id']; ?>">
    <button type="submit" class="btn-delete">Удалить</button>
</form>
```

### Методы защиты
1. **CSRF токены** - уникальные случайные значения для каждой сессии
2. **POST для изменяющих операций** - GET только для чтения
3. **hash_equals()** - защита от timing атак при сравнении токенов
4. **SameSite cookie** - дополнительная защита на уровне браузера

## 4. Information Disclosure

### Описание уязвимости
Утечка чувствительной информации через сообщения об ошибках, заголовки HTTP или другие каналы.

### Найденные уязвимости

#### До исправления:
```php
// Подробные ошибки PDO видны пользователю
$db = new PDO($dsn, $user, $pass);
// При ошибке SQL выводится полный текст запроса и структура БД

// Небезопасное сравнение паролей
if ($_SERVER['PHP_AUTH_PW'] == '123') {
    // Уязвимо к timing атакам
}
```

#### После исправления:

**Отключение вывода ошибок PDO:**
```php
// save.php, admin.php, edit.php
try {
    $db = new PDO($dsn, $user, $pass, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
    ]);
} catch (PDOException $e) {
    // В production не показываем детали ошибки
    error_log('Database connection failed: ' . $e->getMessage());
    die('Ошибка подключения к базе данных.');
}
```

**Защита от timing атак (admin.php):**
```php
// До - уязвимый код
if ($_SERVER['PHP_AUTH_PW'] != '123') {
    // Время выполнения зависит от позиции первого несовпадающего символа
}

// После - безопасный код
if (!hash_equals(md5('123'), md5($_SERVER['PHP_AUTH_PW']))) {
    // Время выполнения постоянно независимо от ввода
    header('HTTP/1.1 401 Unauthorized');
    header('WWW-Authenticate: Basic realm="Admin Panel"');
    exit('401 Требуется авторизация');
}
```

**Регенерация Session ID (login.php):**
```php
// После успешной авторизации
if ($user && hash_equals($user['password'], $password_hash)) {
    // Регенерируем session ID для защиты от session fixation
    session_regenerate_id(true);
    
    $_SESSION['login'] = $login;
    $_SESSION['logged_in'] = true;
    
    header('Location: form.php');
    exit();
}
```

### Методы защиты
1. **PDO::ERRMODE_EXCEPTION** - контролируемая обработка ошибок
2. **error_log()** - логирование ошибок вместо вывода пользователю
3. **hash_equals()** - constant-time сравнение для защиты от timing атак
4. **session_regenerate_id()** - защита от session fixation

## 5. File Inclusion

### Описание уязвимости
Уязвимости типа Local/Remote File Inclusion позволяют злоумышленнику включить произвольные файлы через параметры запроса.

### Анализ кода

#### Проверка всех файлов:
```bash
# Поиск потенциально опасных конструкций
grep -n "include\|require" task3/*.php
```

**Результат:** В проекте используются только статические include/require:
```php
// security.php
require_once 'security.php';  // Статический путь - безопасно

// Нет динамических включений типа:
// include($_GET['page'] . '.php');  // ОПАСНО - не используется
```

### Методы защиты
1. **Статические пути** - все include/require используют фиксированные пути
2. **Отсутствие пользовательского ввода** в путях к файлам
3. **Если бы требовалось динамическое включение:**
```php
// Правильный подход - whitelist
$allowed_pages = ['home', 'about', 'contact'];
$page = $_GET['page'] ?? 'home';

if (!in_array($page, $allowed_pages, true)) {
    $page = 'home';
}

include $page . '.php';
```

**Статус:** ✅ Уязвимости не обнаружены

## 6. File Upload

### Описание уязвимости
Небезопасная загрузка файлов может позволить злоумышленнику загрузить и выполнить вредоносный код на сервере.

### Анализ кода

#### Проверка функционала загрузки:
```bash
# Поиск обработки файлов
grep -n "\$_FILES\|move_uploaded_file" task3/*.php
```

**Результат:** В проекте отсутствует функционал загрузки файлов.

### Рекомендации на будущее

Если потребуется добавить загрузку файлов (например, аватаров), использовать:

```php
// Пример безопасной загрузки изображений
if (isset($_FILES['avatar'])) {
    $file = $_FILES['avatar'];
    
    // 1. Проверка ошибок загрузки
    if ($file['error'] !== UPLOAD_ERR_OK) {
        die('Ошибка загрузки файла');
    }
    
    // 2. Проверка размера (макс 2MB)
    if ($file['size'] > 2 * 1024 * 1024) {
        die('Файл слишком большой');
    }
    
    // 3. Проверка MIME типа
    $allowed_types = ['image/jpeg', 'image/png', 'image/gif'];
    $finfo = finfo_open(FILEINFO_MIME_TYPE);
    $mime = finfo_file($finfo, $file['tmp_name']);
    finfo_close($finfo);
    
    if (!in_array($mime, $allowed_types, true)) {
        die('Недопустимый тип файла');
    }
    
    // 4. Проверка расширения
    $ext = strtolower(pathinfo($file['name'], PATHINFO_EXTENSION));
    if (!in_array($ext, ['jpg', 'jpeg', 'png', 'gif'], true)) {
        die('Недопустимое расширение файла');
    }
    
    // 5. Генерация безопасного имени файла
    $new_name = bin2hex(random_bytes(16)) . '.' . $ext;
    
    // 6. Сохранение вне webroot или с защитой .htaccess
    $upload_dir = __DIR__ . '/uploads/';
    if (!is_dir($upload_dir)) {
        mkdir($upload_dir, 0755, true);
    }
    
    // 7. Перемещение файла
    if (!move_uploaded_file($file['tmp_name'], $upload_dir . $new_name)) {
        die('Не удалось сохранить файл');
    }
    
    // 8. Сохранение пути в БД
    $stmt = $db->prepare("UPDATE users SET avatar = :avatar WHERE id = :id");
    $stmt->execute([':avatar' => $new_name, ':id' => $user_id]);
}

// .htaccess в папке uploads для защиты от выполнения PHP
/*
<FilesMatch "\.php$">
    Order Allow,Deny
    Deny from all
</FilesMatch>
*/
```

### Методы защиты
1. **Whitelist MIME типов** - проверка через finfo_file()
2. **Whitelist расширений** - только разрешенные форматы
3. **Ограничение размера** - защита от DoS
4. **Случайные имена файлов** - предотвращение перезаписи
5. **Хранение вне webroot** или с .htaccess защитой
6. **Проверка содержимого** - для изображений можно использовать getimagesize()

**Статус:** ✅ Функционал отсутствует, рекомендации подготовлены

## 7. Дополнительные улучшения безопасности

### 7.1 Безопасность сессий

#### Настройка cookie параметров:
```php
// Все файлы с сессиями
session_start([
    'cookie_httponly' => true,  // Защита от XSS - JS не может прочитать cookie
    'cookie_secure' => true,    // Только HTTPS (для production)
    'cookie_samesite' => 'Strict'  // Защита от CSRF
]);
```

**Эффект:**
- `HttpOnly` - JavaScript не может украсть session cookie через XSS
- `Secure` - cookie передается только по HTTPS
- `SameSite=Strict` - cookie не отправляется при переходе с других сайтов

### 7.2 Валидация данных

#### Whitelist валидация языков программирования:
```php
// save.php, edit.php
$allowed_languages = ['php', 'javascript', 'python', 'java', 'csharp', 'ruby'];
$languages = isset($_POST['languages']) ? (array)$_POST['languages'] : [];

// Оставляем только разрешенные значения
$languages = array_intersect($languages, $allowed_languages);
```

#### Валидация года рождения:
```php
// save.php
$year = filter_input(INPUT_POST, 'year', FILTER_VALIDATE_INT);
if ($year === false || $year < 1900 || $year > date('Y')) {
    $errors[] = 'Введите корректный год рождения.';
}
```

#### Валидация пола:
```php
// save.php
$allowed_genders = ['male', 'female'];
if (!in_array($gender, $allowed_genders, true)) {
    $errors[] = 'Выберите пол.';
}
```

### 7.3 Защита от перебора паролей

#### Rate limiting (рекомендация):
```php
// Пример для login.php - ограничение попыток входа
session_start();

$max_attempts = 5;
$lockout_time = 900; // 15 минут

if (!isset($_SESSION['login_attempts'])) {
    $_SESSION['login_attempts'] = 0;
    $_SESSION['last_attempt'] = time();
}

// Сброс счетчика после lockout_time
if (time() - $_SESSION['last_attempt'] > $lockout_time) {
    $_SESSION['login_attempts'] = 0;
}

// Проверка количества попыток
if ($_SESSION['login_attempts'] >= $max_attempts) {
    $wait_time = $lockout_time - (time() - $_SESSION['last_attempt']);
    die("Слишком много попыток входа. Попробуйте через " . ceil($wait_time / 60) . " минут.");
}

// При неудачной попытке
if (!$user || !hash_equals($user['password'], $password_hash)) {
    $_SESSION['login_attempts']++;
    $_SESSION['last_attempt'] = time();
    $errors[] = 'Неверный логин или пароль.';
}

// При успешном входе
$_SESSION['login_attempts'] = 0;
```

### 7.4 Content Security Policy

#### Рекомендация для всех страниц:
```php
// Добавить в начало каждого PHP файла
header("Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';");
header("X-Content-Type-Options: nosniff");
header("X-Frame-Options: DENY");
header("X-XSS-Protection: 1; mode=block");
```

**Эффект:**
- CSP ограничивает источники загрузки ресурсов
- `nosniff` предотвращает MIME-sniffing атаки
- `X-Frame-Options` защищает от clickjacking
- `X-XSS-Protection` включает встроенный XSS фильтр браузера

## Рекомендации для production

1. **HTTPS обязателен** - включить `cookie_secure => true`
2. **Отключить display_errors** в php.ini
3. **Настроить error_log** для записи ошибок в файл
4. **Использовать переменные окружения** для паролей БД
5. **Регулярно обновлять PHP** и зависимости
6. **Настроить CSP заголовки**
